# 📱 Shipping EGG FRYER 3000 to the App Store

The repo is pre-wired for [Capacitor](https://capacitorjs.com), which wraps the
game in a native iOS shell (WKWebView) with the web files bundled *inside* the
app — it runs fully locally, no website involved, which is what Apple's
review guideline 4.2 wants to see.

## What you need

- **A Mac with Xcode 16+** (free from the Mac App Store) — Apple requires
  building and signing on macOS, no exceptions
- **Apple Developer Program** membership — $99/year at
  [developer.apple.com](https://developer.apple.com/programs/) — needed for
  TestFlight and the App Store
- Node 18+ on the Mac

## One-time setup (on the Mac)

```bash
git clone https://github.com/baillmarc-sketch/eggs.git
cd eggs
npm install
npm run build:www        # stages the game files into www/
npx cap add ios          # generates the native Xcode project in ios/
npm run ios:assets       # generates all icon + splash sizes from resources/
npm run ios:open         # opens the project in Xcode
```

In Xcode, one-time config:

1. Select the **App** target → **Signing & Capabilities** → choose your Team
   (your Apple Developer account). Xcode handles certificates automatically.
2. Check the bundle ID is `com.baill.eggfryer3000` (set in
   `capacitor.config.json` — change it there if you prefer another).
3. Pick your iPhone (plugged in) or a simulator and press **Run** ▶️ to play it
   as a native app.

## Each release after that

```bash
git pull
npm run ios:sync         # rebuild www/ and copy into the iOS project
```

Then in Xcode: **Product → Archive** → **Distribute App** → **App Store
Connect** → Upload. The build appears in
[appstoreconnect.apple.com](https://appstoreconnect.apple.com) a few minutes
later.

## TestFlight (recommended first stop)

In App Store Connect → your app → **TestFlight**: add family testers by email
(up to 100 external testers). They install via the TestFlight app — feels
exactly like a real install, no App Review needed for internal testers, and a
lightweight review for external ones. Many family apps simply live here.

## Full App Store release

In App Store Connect → **App Store** tab:

- **Screenshots**: 6.7" and 6.1" iPhone sizes required (take them in the
  Xcode simulator: `⌘S` saves a correctly-sized screenshot)
- **Privacy**: declare "Data Not Linked to You → User Content (name, score)"
  — the leaderboard stores a typed name and a score in Firebase, nothing else;
  no tracking, no ads
- **Age rating**: everything "No" → 4+
- **Category**: Games → Casual or Family

Submit for review. Casual games with bundled content typically pass in a
day or two.

## Notes

- `www/` and `ios/` are gitignored — `www/` is generated by `npm run
  build:www`, and the native project stays on your Mac (commit it from there
  if you ever want CI builds).
- The leaderboard keeps working unchanged inside the app — it's plain HTTPS
  to Firebase, and the nightly 2am reset stays on GitHub Actions.
- `@capacitor/haptics` is included in case you want native haptic feedback
  later (the web vibration API is a no-op on iOS).
